1. Project Overview

VisGait is an interactive React and D3.js-based data visualization dashboard developed for the DH2321 course at KTH.

By translating complex raw sensor data from knee braces into high-level, readable visual indicators, VisGait bridges the gap between raw biomedical data and actionable health insights. The dashboard is split into two distinct views: one empowering patients to intuitively track their recovery and customize their activity goals, and another enabling researchers to analyze cohorts, find statistical correlations, and explore distributions of gait anomalies.

2. How To Use (Munzner's Framework)

User I: Patients

The Patient Dashboard prioritizes clarity, motivation, and personalization.

  • Task 1 (Monitoring General Recovery):
    Action: Analyze (Discover) / Search (Explore) | Target: Trends.
    Idiom: Weekly Progress Line Chart & Health Bar. Patients evaluate a single aggregate "Composite Score" over 12 weeks to see if they are improving and staying in the "Healthy Zone".
  • Task 2 (Identifying Suitable Activities):
    Action: Search (Locate) / Query (Compare) | Target: Extremes / Features.
    Idiom: Activities Overview (Radar Chart) & Weekly Bar Charts. Allows patients to quickly compare their performance across 4 specific activities (Stair Climbing, Sit-To-Stand, Timed Up-and-Go, Walking).
  • Task 3 (Personalizing Goals):
    Action: Produce (Derive).
    Idiom: Custom Composite Weighting Slider. An interactive multi-handle slider allowing patients to redefine their overall score based on the daily activities that matter most to them.

User II: Researchers

The Researcher Dashboard allows for deep cohort analysis and multidimensional correlation.

  • Task 1 (Comparing Cohorts & Multi-attribute Analysis):
    Action: Analyze (Discover) / Query (Compare) | Target: Distribution / Correlation.
    Idiom: Parallel Coordinates Plot. Researchers filter and trace patients grouped by recovery status (Improving, Declining, Stable) across multiple variables (GSI, Symmetry Ratio, Cadence, Step Time, etc.).
  • Task 2 (Finding Metric Correlations):
    Action: Search (Explore) | Target: Correlation.
    Idiom: Scatter Plot Matrix. Generates Pearson's r correlation grids to uncover hidden biomechanical relationships between metrics.
  • Task 3 (Evaluating Distribution & Variance):
    Action: Query (Summarize) | Target: Distribution / Trends.
    Idioms: Violin Plot & Line Plot with Standard Deviation. Replaces traditional box plots to better visualize the underlying probability density of metrics across cohorts over time.

Limitations

  • Synthetic Demonstration Data: The gait performance data currently visualized in the web prototype is partly synthetic and generated strictly for UI/UX demonstration purposes.
  • : Peaks from the raw signals of the knee braces still need to be labelled either manually or by algorithms, and further processed to give the metrics for visualization.
  • Web Responsiveness: Dense multivariate charts (like the Scatter Plot Matrix and Parallel Coordinates) scale poorly on smaller mobile devices.
